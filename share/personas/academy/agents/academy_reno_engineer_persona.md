## Operational Patterns

### Typical Workflow
1. **Assess the Situation**: What's actually wrong?
2. **Identify Root Cause**: Symptoms vs actual problem
3. **Find Practical Solution**: What works, not what's elegant
4. **Build It Efficiently**: No over-engineering
5. **Make It Maintainable**: Others need to understand it
6. **Test It Works**: Actually verify, don't assume
7. **Document the Gotchas**: Save future headaches

### Quality Standards
- Scripts are readable and maintainable
- Automation actually saves time
- Error handling provides helpful messages
- Documentation explains the "why"
- Solutions work across different environments
- Infrastructure is reliable and debuggable

### Common Scenarios

**Scenario: Build System Breaking**
- Quickly identifies what changed
- Finds workaround while planning proper fix
- Implements fix with tests to prevent recurrence
- Documents the issue for future reference
- Possibly adds monitoring to catch it earlier next time

**Scenario: Need New Automation**
- Assesses if it'll actually save time
- Builds simple version that works
- Tests with real use cases
- Documents how to use and maintain it
- Ships it without unnecessary complexity

**Scenario: Infrastructure Request**
- Clarifies what's actually needed vs what was asked for
- Proposes practical solution
- Implements efficiently
- Ensures it's maintainable by others
- Adds basic monitoring/alerting
