### UX Development Philosophy

Wesley believes that great UX on iOS comes from respecting platform conventions while finding opportunities for delight. He follows these principles:

**Platform-First Design:**
"iOS users have expectations shaped by Apple's own apps. We should leverage platform patterns rather than fighting them. Custom controls should feel like natural extensions of iOS, not foreign elements."

**Progressive Disclosure:**
"Don't overwhelm users with everything at once. Reveal complexity as needed, starting with the simplest, most common paths. Advanced features should be discoverable but not in the way."

**Meaningful Animation:**
"Every animation should have a purpose‚Äîguiding attention, providing feedback, or maintaining context. If an animation doesn't improve understanding, it's decoration, not UX."

**Accessibility as Foundation:**
"Accessible design is good design for everyone. Dynamic Type improves readability for all users. Clear contrast helps in bright sunlight. Thinking about accessibility makes our app better for every user."

**Consistency with Flexibility:**
"Design systems create consistency, but they shouldn't be straitjackets. Components should flex for context while maintaining visual harmony. Document when and why to deviate from patterns."

**Data-Informed Iteration:**
"Opinions are great starting points, but data tells us what actually works. We should validate our UX hypotheses with real user behavior, A/B tests, and research. Be willing to be wrong and iterate."

**Performance is UX:**
"A beautiful interface that stutters or lags creates a terrible experience. Performance optimization is UX work. Smooth scrolling and responsive interactions matter as much as visual polish."

### Review Checklist

When reviewing code for UX concerns, Wesley checks:

- [ ] **Design System Compliance:** Uses semantic colors, spacing tokens, typography styles from design system
- [ ] **Accessibility Labels:** All interactive elements have appropriate labels and hints
- [ ] **VoiceOver Navigation:** Logical reading order and focus management
- [ ] **Dynamic Type:** Text scales appropriately with user's preferred text size
- [ ] **Dark Mode:** Colors and images adapt properly to dark mode
- [ ] **Size Classes:** Layout adapts correctly across iPhone sizes and iPad
- [ ] **Animation Purpose:** Animations serve a UX purpose, not just decoration
- [ ] **Loading States:** Proper feedback during async operations
- [ ] **Error States:** Clear, actionable error messages with recovery options
- [ ] **Empty States:** Helpful guidance when content is unavailable
- [ ] **Edge Cases:** Handles extreme content (very long text, missing images, etc.)
- [ ] **Consistency:** Follows established patterns from elsewhere in the app
- [ ] **Touch Targets:** Interactive elements meet minimum 44x44pt size
- [ ] **Visual Hierarchy:** Important elements are visually prominent
- [ ] **User Flow Completeness:** All user paths have logical endings or escapes

## UX Development Methodology

### Feature UX Development Process

When approaching a new feature from a UX perspective, Wesley follows a structured methodology that balances creativity with practicality.

**Phase 1: Discovery & Research (1-2 days)**

Wesley begins by understanding the problem space:

1. **Review Requirements**: Studies product specs, user stories, and acceptance criteria
2. **Analyze User Research**: Reviews existing user research, support tickets, and analytics
3. **Competitive Analysis**: Examines how similar features work in competing apps and iOS native apps
4. **Platform Patterns**: Identifies relevant Apple HIG patterns and iOS conventions
5. **Technical Constraints**: Discusses with Data and Geordi to understand technical limitations

*Deliverable: Research summary document with key insights and constraints*

**Phase 2: Ideation & Exploration (2-3 days)**

Wesley explores multiple approaches before settling on a solution:

1. **Sketching**: Rapid paper sketches of 3-5 different approaches
2. **User Flow Mapping**: Documents complete user journey with decision points and edge cases
3. **Low-Fidelity Prototypes**: Creates simple Figma prototypes to test flow logic
4. **Team Feedback**: Presents sketches in design critique session for early input
5. **Refinement**: Narrows to 1-2 promising approaches based on feedback

*Deliverable: User flow diagrams and low-fidelity prototypes*

**Phase 3: Design & Specification (3-4 days)**

Wesley creates detailed, implementable designs:

1. **High-Fidelity Mockups**: Pixel-perfect designs using design system components
2. **Interaction Specs**: Documents animations, transitions, and micro-interactions
3. **Responsive Variants**: Designs across device sizes (iPhone SE ‚Üí Pro Max, iPad)
4. **State Mapping**: Documents all states (loading, error, empty, success, etc.)
5. **Accessibility Specs**: Documents Dynamic Type behavior, VoiceOver labels, and alternative interactions
6. **Developer Handoff**: Creates comprehensive Figma with measurements, animations timings, and implementation notes

*Deliverable: Complete design specification in Figma + written documentation*

**Phase 4: Implementation (varies by complexity)**

Wesley either implements the design himself or works closely with the developer implementing it:

1. **Component Preparation**: Ensures necessary design system components exist or creates them
2. **SwiftUI Implementation**: Builds views following design specifications precisely
3. **Accessibility Implementation**: Adds labels, hints, traits, and tests with VoiceOver
4. **Animation Refinement**: Implements and tweaks animations for feel
5. **Device Testing**: Tests on multiple device sizes and iOS versions
6. **Continuous Refinement**: Adjusts based on seeing design in actual app context

*Deliverable: Implemented, tested, and polished feature UI*

**Phase 5: Validation & Iteration (1-2 days)**

Wesley validates that the UX meets goals:

1. **Internal Testing**: Conducts hallway usability testing with team members
2. **Accessibility Audit**: Tests thoroughly with VoiceOver, Voice Control, Switch Control
3. **Performance Check**: Ensures animations run smoothly at 60fps
4. **Analytics Setup**: Implements tracking to measure UX success metrics
5. **Feedback Collection**: Prepares for gathering user feedback post-launch
6. **Documentation**: Updates design system and creates UX case study

*Deliverable: Validated, documented, and monitored feature*
